# Starblox

**Describe a game in plain words ‚Üí an AI builds it ‚Üí play it with friends in
seconds ‚Üí reshape it mid-match, just by asking.** A multiplayer game platform
built on [SpacetimeDB](https://spacetimedb.com), with a host-authoritative
"pseudo-engine" that runs AI-authored games.

> **Status: working MVP / proof-of-concept.** The pieces below are real and
> wired. The roadmap ([PROJECT.md](PROJECT.md)) is turning the pseudo-engine into
> a full game engine ‚Äî read "What's real today" before reading more into it.

---

## What's real today

- **The engine.** A generic, host-authoritative runtime (`src/engine/`): one
  elected client runs a 30 Hz tick, commits the entity set to SpacetimeDB, and
  every client renders from the synced state. Two reference games
  (`src/games/flappy.ts`, `tank.ts`) prove the SDK can express real multiplayer
  games. Validated by two-browser integration tests (`e2e/engine*.spec.ts`).
- **The generation pipeline.** `POST /api/create` turns a sentence into a runnable
  game module, generated by Claude against the engine's SDK contract
  (`src/lib/gamegen.ts`). A loader (`src/engine/loader.ts`) compiles that code and
  the engine runs it ‚Äî generated games and the reference games take the same path.
- **Live editing.** While a game runs, you describe a change and it applies for
  everyone, mid-match, no reload (`/api/edit` for rules games; `EngineEditChat`
  + live `engine_config` for engine games).
- **Multiplayer, instantly.** Every room is a link + QR; friends join and play.
  No installs, no accounts.

## Demo mode vs. live mode ‚Äî read this

The AI runs in one of two modes, chosen by whether an API key is present. This is
the single most important thing to understand about the repo:

| | **Demo mode** (no key) | **Live mode** (`ANTHROPIC_API_KEY` set) |
|---|---|---|
| `/api/create`, `/api/edit` | return a **captured output** of the pipeline, tagged `source: "cached"` | call **Claude** and generate/edit for real |
| Why | offline, deterministic, stage-safe | the real product |

**The cached responses are snapshots of this exact pipeline's output** ‚Äî the seed
reference games and pre-validated edits ‚Äî not stand-ins for something that doesn't
exist. The generation / loader / sandbox code is right here in `src/`; a key turns
it on. It runs in cached mode for reliability and never pretends to be live when it
isn't. To run it live, add to `.env.local`:

```
ANTHROPIC_API_KEY=sk-ant-...
# ANTHROPIC_MODEL=claude-opus-4-8   # optional override
```

## Run it locally

1. **Backend:** `spacetime start`, then `npm run stdb:publish` (publishes the
   `blox` module) and `npm run stdb:generate`.
2. **Frontend:** `npm run dev` ‚Üí http://localhost:3001 (`.env.local` defaults to
   the local backend; `NEXT_PUBLIC_TEST_MODE=1` enables demo helpers).
3. **Tests:** `npm run test:e2e` (needs the local backend running).

Deploying to production: [DEPLOY.md](DEPLOY.md).

## Repo map

- `src/engine/` ‚Äî the runtime, SDK types, **loader + sandbox**.
- `src/games/` ‚Äî the reference game modules (flappy, tank).
- `src/lib/gamegen.ts` ‚Äî the generation contract (the SDK the model targets).
- `src/lib/buildAgent.ts` ‚Äî the generate ‚Üí compile-check ‚Üí fix loop.
- `src/app/api/create`, `src/app/api/edit` ‚Äî the AI endpoints (live or cached).
- `spacetimedb/src/index.ts` ‚Äî the SpacetimeDB module (tables + reducers).
- `e2e/` ‚Äî Playwright integration tests (the real multiplayer proofs).
